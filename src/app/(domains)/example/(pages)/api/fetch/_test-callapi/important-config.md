## 🎯 핵심 포인트

### 일반 웹 vs 금융권 차이

| 설정                     | 일반 웹 | 금융권    | 이유             |
| ------------------------ | ------- | --------- | ---------------- |
| **staleTime**            | 60초    | **0초**   | 항상 최신 데이터 |
| **gcTime**               | 5분     | **0초**   | 민감정보 보호    |
| **refetchOnWindowFocus** | false   | **true**  | 탭 전환 시 갱신  |
| **retry (mutation)**     | 3       | **false** | 중복 거래 방지   |

### 💡 가장 중요한 것

```typescript
// ❌ 절대 하면 안 되는 것
mutations: {
	retry: 3; // 이체/결제가 3번 실행될 수 있음!
}

// ✅ 반드시 해야 하는 것
mutations: {
	retry: false; // 거래는 한 번만!
}
```

### 📊 데이터 유형별 추천 설정

1. **환율/주가** (실시간) → staleTime: 0, refetchInterval: 5초
2. **은행코드** (정적) → staleTime: 1시간, 캐싱 적극 활용
3. **잔액** (중요) → staleTime: 0, refetchOnWindowFocus: true
4. **거래내역** (조회) → staleTime: 1분, 적당한 캐싱

### ⚠️ 보안 필수사항

- 페이지 닫을 때 캐시 전체 삭제
- 401 에러 시 즉시 로그아웃
- 모든 거래는 감사 로그 남기기

금융권에서는 **성능보다 정확성과 보안이 우선**입니다! 🔒
